Key Implementation Details:

- Event objects use Win32 CreateEvent for synchronization
- WaitForGpu() checks fence values before waiting to avoid unnecessary blocks
- Execute() method handles command list execution but currently doesn't flush (for testing)
- Proper resource cleanup in destructor

CommandQueue Architecture:
- Initialize(): Creates command queue, fence, and event objects
- Execute(): Executes command lists on the GPU
- Flush(): Signals fence and waits for GPU completion
- Signal(): Sets fence value on GPU
- WaitForGpu(): Waits for GPU to reach specified fence value
- GetCommandQueue(): Provides access to underlying ID3D12CommandQueue

Synchronization Flow:
1. CPU submits command lists via Execute()
2. CPU signals fence value via Signal()
3. CPU checks current GPU fence value
4. If GPU hasn't reached signalled value, CPU waits on event
5. Event fires when GPU completes work and updates fence value
